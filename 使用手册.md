# 试纸色差检测系统 使用手册

## 环境准备

### 创建虚拟环境

```bash
# 在项目根目录下创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate    # macOS / Linux
# venv\Scripts\activate     # Windows
```

### 依赖安装

```bash
# 确保已激活虚拟环境后执行
pip install -r backend/requirements.txt
```

### 启动服务

**终端 1 — 后端：**
```bash
source venv/bin/activate
cd backend
python3 app.py
```
启动后显示 `Running on http://0.0.0.0:5002`

**终端 2 — 前端：**
```bash
cd frontend
python3 -m http.server 8080
```

打开浏览器访问：`http://localhost:8080`

## 操作步骤

### 第一步：设置参数

**ΔE 阈值**（默认 5）
- 拖动滑块调整，范围 0~30
- 含义：网格色差超过此值则判定为变色
- 建议：根据试纸灵敏度调整，一般 3~10 之间

**网格数量**（默认 50）
- 输入框直接填写，范围 3~200
- 含义：将试纸划分为多少个网格进行统计
- 建议：50 是精度与速度的平衡点，精度要求高可设 100

### 第二步：上传图片

- 点击上传区域，或将图片**拖拽**到上传区
- 支持格式：JPG、PNG
- 上传成功后显示图片预览
- 如需更换图片，点击预览图即可重新上传

> **拍摄建议**：
> - 试纸平放，避免弯曲
> - 光线均匀，避免强烈阴影
> - 试纸占画面 60% 以上
> - 确保定位孔（黑洞）清晰可见

### 第三步：开始分析

点击「开始分析」按钮，等待进度条完成（通常 2~5 秒）。

## 查看结果

### 步骤条

分析完成后，顶部出现 4 个步骤节点：

| 步骤 | 内容 |
|------|------|
| Step 1 ROI 提取 | 查看试纸轮廓检测和校正结果 |
| Step 2 黑洞裁剪 | 查看定位孔识别和裁剪效果 |
| Step 3 分割与色差 | 查看左右分割线、LAB 均值、整体 ΔE |
| Step 4 网格统计 | 查看热力图和变色标注图 |

- 点击任意步骤节点可跳转查看
- 也可点击「上一步 / 下一步」按钮顺序浏览

### 结果卡片（底部）

| 卡片 | 说明 |
|------|------|
| 左侧平均 LAB | 对照区的 L、a、b 值，作为基准参考色 |
| 右侧平均 LAB | 反应区的 L、a、b 值 |
| 整体 ΔE | 左右均值的色差，反映整体反应程度 |
| 超阈值比例 | 变色网格数 ÷ 总网格数，反映变色面积占比 |

## 结果解读

### ΔE 参考范围

| ΔE 值 | 人眼感知 | 含义 |
|-------|---------|------|
| 0~1 | 几乎无差异 | 对照区与反应区颜色基本一致，未反应 |
| 1~3 | 轻微差异 | 轻微变色，反应较弱 |
| 3~6 | 明显差异 | 明显变色，反应中等 |
| > 6 | 显著差异 | 显著变色，反应强烈 |

### 变色比例解读

- **比例 = 超阈值网格数 ÷ 全图总网格数**
- 分子：右侧（反应区）中 ΔE 超过阈值的网格数
- 分母：左右两侧全图总网格数
- 例：比例 24% 表示全图约 1/4 的区域发生了超阈值变色

### 热力图颜色说明

- 蓝色：ΔE 低，颜色接近对照区
- 红色：ΔE 高，颜色偏离对照区明显
- 变色标注图中红色高亮区域为超过阈值的网格

## 常见问题

**Q：分析失败，提示"未检测到试纸区域"**
- 检查图片是否清晰，试纸是否占画面主体
- 尝试在光线更均匀的环境下重新拍摄

**Q：黑洞没有被正确识别，裁剪位置不对**
- 确保定位孔（黑洞）在图片中清晰可见，颜色足够深
- 检查图片是否过曝或欠曝

**Q：变色比例异常偏高或偏低**
- 调整 ΔE 阈值：阈值越小，判定为变色的网格越多；阈值越大，越少
- 增大网格数量可提高统计精度

**Q：后端启动报错 `Address already in use`**
- 5002 端口被占用，执行以下命令杀掉占用进程：
```bash
kill -9 $(lsof -ti:5002)
```
